O Básico

Vamos começar explicando algumas "siglas" e termos:
  • World Wide Web (www, ou web) = um sistema de hipermídia que atua no mundo inteiro, daí vem o nome (em português: "teia em todo o mundo");
  • Hipermídia = sistema de informações informatizadas, como documentos, textos, vídeos, áudios e etc, que por meio de link acionam outras funções e assim sucessivamente;
  • Hipertexto = informações de textos organizadas de forma que o usuário pode decidir o caminho em que ele percorrerá;
  • URL = Endereço único da página na web, é por meio da URL que o usuário vai encontrar a página;
  • HTTP (HyperText Transfer Protocol) = protocolo padrão da web para transferências de dados e requisições, resumindo é a forma de transferir dados do cliente para o servidor, porém qualquer usuário mal intencionado pode interceptar os dados, pegando, lendo ou até mesmo modificandos eles;
  • HTTPS = é o HTTP "seguro", há alguns princípios de segurança adicionados, como criptografia (os dados não transitam da forma "original", e sim modificados com uma criptografia), confidencialidade (a mensagem só é lida pelo real destinatário), integridade (a mensagem não é modificada) e autenticação (a autenticação prova que o servidor é realmente quem diz ser);
  • HTML (HyperText Markup Language) = é uma linguagem de formatação de hipertexto, ou seja é o método de codificar as informações e as exibi-la de formas diferentes;
  • CSS (Cascading Style Sheets) = podemos dizer que é o irmão do HTML, os dois trabalham juntos quase obrigatoriamente, o CSS é responsável pela estilização das páginas HTML;
  • Javascript (JS) = linguagem de programação presente em quase tudo na web, pode ser usado tanto no back-end quanto no front-end;
  • Bibliotecas = é uma forma de compartilhar e disponibilzar soluções por meio de códigos (funções e métodos), exemplo: você vai em uma biblioteca e pega um livro "físico" de matemática e utiliza as equações, ou seja, você não precisou desenvolve-lás, apenas utilizá-las, uma verdadeira mão na roda!

  • Frameworks = conjunto de códigos desenvolvidos em um conjunto que é utilizado para resolver problemas e facilitar algumas funções, um framework te disponibilza as ferramentas, te auxília no uso e padroniza os recursos.
  • APIs (Application Programming Interface) = As APIs permitem que o desenvolvedor acesse funcionalidades de aplicações já existentes, exemplo: você pode utilizar a API do Whatsapp ou do Twiter e etc.
  • Banco de Dados = Organização de dados concentrados em um lugar utilizados para fazer "pesquisas" dentro do sistema
  • Back-end = códigos utilizados para fazer a parte que o usuário não vê, é a parte que faz as coisas funcionarem;
  • Front-end = códigos utilizados para fazer tudo que voçê vê, textos, imagens, cores, vídeos e etc;
  • Full-stack = programador que programa Back-end e Front-end;
  • JSON (JavaScript Object Notation) = formatação de texto utilizado para transferir dados facilmente, é um texto legível para humanos e para a web;
  • IDEs (Integrated Development Environment) = em português IDE significa: Ambiente de Desenvolvimento Integrado, é basicamente um conjunto de ferramentas com um editor de texto que permite o desenvolvimento de códigos com maior eficácia e velocidade.
  • Programas Open Source = Programas Open Source ou, em português, programas de Código Aberto, são basicamente programas que tem seu código fonte disponibilizado e licenciado com uma liçenca na qual você pode editar e modificar oque quiser e como quiser e distribuir o programa de graça para quem quiser.

O que é ser um desenvolvedor web?

O papel principal de um desenvolvedor web, sem dúvidas, é criar sites e ter certeza de que tudo que está lá vai funcionar independente das circunstâncias, exemplo: mesmo se o usuário acessar os site em um dispositivo mobile ou em um computador ele deve funcionar corretamente.


Um desenvolvedor web ou no inglês, web developer, pode se dividir em três áreas de trabalho:


  • Front-end: como já explicado é quem trabalha na parte visual;
  • Back-end: como também já explicado, é quem trabalha na parte que o usuário não vê;
  • Full-stack: novamente explicando, é quem trabalha em ambas as partes de um website;

O que um desenvolvedor Front-end precisa?

O conjunto de linguagens que um desenvolvedor Front-end necessita, resumidamente consiste em HTML, CSS e Javascript, e alguns Frameworks, como, Less, Bootstrap, Boilerplate, e alguns outros.

O que um desenvolvedor Back-end precisa?

O Back-end normalmente gira em torno de banco de dados, então uma das linguagens principais que um desenvolvedor Back-end precisa saber é concerteza SQL, outra linguagem importante, é o PHP, que vai ajudar na interação entre o banco de dados e o site.


Existem várias e várias outras linguagens e ferramentas voltadas a web que são bem importantes, exemplo: Node.Js, Python, e sem contar o Javascript que funciona em Full-stack.

Conceitos importantes para um desenvolvedor Back-end

Servidor: arquitetura cliente-servidor; um servidor pode ser definido como algo que executa funções centralizadas em um local, cliente é quem acessa o servidor e utiliza os dados/funções, por exemplo você que está no seu computador e acessa a internet pelo seu navegador (Chrome, Firefox, Edge, Opera e etc.);

Database (banco de dados): como explicado resumidamente acima, um banco de dados armazena dados, e saber sobre banco de dados é algo essencial, você certamente irá ter que aprender ferramentas como MySQL ou MongoDB ou Postgresql;

API: como também já explicado acima, APIs te disponibilizam funções de aplicações já existentes, o conhecimento delas irão te ajudar muito;

Flexibilidade: saber se flexionar é muito importante, a tecnologia está sempre mudando, então é essencial estar apto para aprender novas linguagens e ferramentas;

Segurança: o desenvolvedor Back-end também tem que se atentar e muito ás práticas de segurança, aliás você não iria querer ter um site invadido.

Ferramentas extras

Outra ferramenta que pode te ajudar no futuro é o Git, que é um sistema de controle de versão de seu software ou projeto, ele registra o histórico de todas as alterações e mudanças feitas na aplicação, aqui tem um bom curso básico pra quem quer aprender um pouco de Git: https://www.youtube.com/watch?v=WVLhm1AMeYE&list=PLInBAd9OZCzzHBJjLFZzRl6DgUmOeG3H0

Se você conhece os sistemas operacionais baseados em Unix sabe que eles já vem com um gerenciador de pacotes implementado, bom primeiro vamos saber oque é um gerenciador de pacotes:


Um gerenciador de pacotes faz pra você todo o processo de instalar um programa e suas dependências e também desinstala-lo com apenas alguns comandos, isso facilita muito, porém o Windows não tem um desses implementado originalmente, diferentemente das versões de Linux, que podem vir com o APT ou YUM.


O melhor gerenciador de pacotes para Windows hoje em dia é o Chocolatey, porém há alguns pré-requisitos:

  • Windows 7 pra cima;
  • Windows Server 2003 pra cima;
  • PowerShell v2 pra cima;
  • .NET Framework 4 pra cima.

Para instalar o Chocolatey na sua máquina siga os passos do próprio tutorial do Chocolatey, se você quiser dar uma olhada em todos os pacotes que o Chocolatey tem, clique aqui: https://chocolatey.org/packages
